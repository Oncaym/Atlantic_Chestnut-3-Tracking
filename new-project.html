🏗 New Tracker Wizard · 建新项目向导

照现有项目建一个新工地的 tracker：core 原样带走，名单换成新项目的。全程本地，不用写代码。
这个浏览器不支持所需能力——请用 Chrome 或 Edge 打开本页。

1选模板项目（比如 AC3 的文件夹）

📁 点这里选择现有 tracker 文件夹

2新项目信息

3粘贴单元清单

每行一个单元，只需要编号（例：SF01）。想带更多信息可选：编号后用 Tab 依次接 描述、区域、楼层、Vision Lites 数、门数、面积 sf、Louver(yes/no)——用不到就留空。表头行自动跳过。

4导入平面图底图（可选，之后随时能换）

默认一个 Ground Floor 输入口；点"添加楼层"可加任意多个自定义楼层（各自命名 + 各自底图）。不选底图就先用模板占位图，之后放进文件夹同名替换即可。

5选目标文件夹并生成

先在电脑上新建一个空文件夹（比如 "AC4 tracker"），然后选它。
📁 点这里选择新建的空文件夹

✓接下来

✅ 只想在本地单机看：双击新文件夹里的 index.html 就能用，下面第 2–6 步全部跳过。
下面这些只有在需要多人云端协作（实时同步、手机现场更新）时才做，约 25 分钟一次性开通。
  1. 双击新文件夹里的 index.html —— 应该能看到全部单元（本地模式横幅是正常的）
  2. 照文件夹里的 SETUP.md 建一个新的 Firebase 项目（20 分钟，不要复用别的项目的）仅云端
  3. 新建 GitHub 仓库并推送，接上 Vercel；仓库 Secrets 里配 FIREBASE_SERVICE_ACCOUNT + FIREBASE_DATABASE_URL（每日备份用），Vercel 环境变量配 ANTHROPIC_API_KEY（Chat 用）仅云端
  4. 把真实平面图换掉占位图（文件名不变），在页面上用 Place mode 摆标记
  5. 对 Claude 说：按新 takeoff 重建 index.html 的 scope 卡片区（那块数量还是模板项目的）
  6. Firebase Console 里给每个 PM 建登录账号仅云端

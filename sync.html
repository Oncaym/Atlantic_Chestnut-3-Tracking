☁ 云端模式：从 GitHub 上的权威 Hub 仓库拉最新 core 文件，并列出 Hub 里你还没有的新功能（读 FEATURES.md）。Hub 仓库需为 public。
📁 本地模式：两个本地文件夹直接对比、复制 core（离线也能用）。
✅ 同步前检查 EDITING.lock，同步时自动上锁、完成删锁、写入后哈希校验。
⛔ 永不触碰项目件：project-config.js、firebase-config.js、state.json、elevations.js、平面图、index.html。
🧠 index.html 是混合件，功能块移植交给 Claude（"把 F-0XX 同步到我的项目"）。想把自己加的功能贡献回 Hub：对 Claude 说"把我的 F-0XX 贡献回 Hub"。
📤 贡献回 Hub：云端模式扫描后，如果你本地的 core 文件比 Hub 新/有分歧、或有 Hub 没有的功能（FEATURES.md 里本地独有的 F-0XX），会出现"📤 贡献回 Hub"按钮。点它会把这些改动文件发到贡献端点，由端点自动开一个 Pull Request；Hub 维护者审核合并后，大家再从云端拉取即可。
🔑 贡献端点需部署为 serverless（api/contribute），并配置环境变量 GITHUB_HUB_TOKEN（细粒度 PAT，仅对 Hub 仓库有 Contents + Pull requests 写权限）。把端点绝对地址填进上面的 contribUrl 栏。
📦 没配 contribUrl 也能用：会退化成下载一个手动打包文件（每个改动文件带 ===== path ===== 分隔头），发给维护者手动开 PR。详见 HUB-SETUP.md。
